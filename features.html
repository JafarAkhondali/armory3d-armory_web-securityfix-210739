Blender Integration

Armory is directly integrated into Blender as an add-on, so there is no need to leave your favorite 3D software to create awesome games. You can use the workflows you are used to without having to constantly switch between different programs.

Blender
Render Path

Armory's render path is designed to work with Blender's material nodes. You can create beautiful materials with ease and choose from a broad range of post-processing effects. Projects can configure multiple render paths so that the optimal one can be selected for the target plaform and hardware. If you need more fine-grained control, you can code your own shaders and even create your own render path from scratch!

Post-Process Materials
Scripting

For creating the interactive behaviour of your game, you can decide between writing Haxe code or using visual scripting in the form of logic nodes. Logic nodes are easy to learn and provide you with a great variety of functionality. Creating your own logic nodes to encapsulate specific functionality is very straightforward.

Logic Nodes
User Interface

Armory comes with its own UI library and a corresponding WYSIWYG editor, making it easy to add user interfaces to your game.

2D Editor
Cross-Platform

Thanks to the underlying Kha framework, Armory runs almost everywhere! Whether you want to target desktop computers, browsers, mobile phones or consoles, Armory can export your game to those platforms. By the way, you can try out the Armory examples and templates directly in your browser: Armory Examples Browser.

Supported Platforms
Physics

Armory makes use of the Bullet physics engine to simulate all kinds of dynamic movements. It can also be configured to use the Oimo physics engine instead.

Notable Features:

  • Rigid body simulation
  • Soft body simulation
  • Ragdoll simulation
  • Ray casting
  • Collision groups
  • Simulation of breaking / fractured meshes

Physics
Navigation

Add realistic NPC behaviour to your game by letting actors move on navigation meshes. Armory uses the Recast navigation system but can be configured to use any other navigation engine.

Navmesh
Particles

Armory supports both emitter and hair particle systems that can be used to create stunning visual effects or grassy landscapes for example. Internally, GPU instancing is used to achieve the best possible performance.

Supported Particles
Animation
  • GPU skinning
  • Keyframes
  • Shape keys
  • Tilesheets

Shape keys
Lighting
  • Lightmapping
  • Area lights using LTC (linearly transformed cosines)
  • Reflection probes
  • IES textures

Lighting
Free & Open-Source

Armory is completely free and its source code is available on Github. Do want to change something specific to your game in the engine? It couldn't be more easy. You don't know how exactly something works and the documentation is unclear? You can easily look it up! Armory uses a permissive license; and compared to another well-known Blender game engine, your games aren't restricted by Blender's GNU GPL license.
